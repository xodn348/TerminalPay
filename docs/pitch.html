termpay

The agentic wallet for AI agents.

A local CLI + MCP server that will let Claude Code, Codex, Cursor, or any MCP-aware agent pay autonomously — within limits the user controls. Card never leaves the machine.

Problem

AI agents can write code, draft emails, run ops — but they can't pay.

Every "top up my Anthropic credits", "renew this domain", "buy this part" forces a human context-switch back into a browser. The agent's autonomy ends at the checkout form.

Existing pathHow it worksWhy it falls short
Stripe ACP / Google AP2 / Coinbase x402 / Visa Trusted Agent Merchant-side protocols. Card or token negotiated over an open standard. Needs merchant adoption. Covers a tiny slice today.
OpenAI Operator / Computer Use raw LLM drives the browser, sees the checkout form, types the card. LLM sees the card. No policy layer. No audit. No kill switch.
Privacy.com virtual cards (manual) Single-use card numbers generated in a web dashboard. No agent integration. The human still pastes the number.
"I'll just paste my card" Human types card into the agent or its system prompt. Card leaks into logs, context windows, third-party LLMs.
Approach

What we're building: a local wallet any MCP-aware agent can call.

The shape

  • Local Node.js CLI + MCP server, runs on the user's machine
  • Card encrypted in ~/.termpay/db.sqlite (AES-256-GCM, key in OS Keychain)
  • MCP tools: pay, purchase, payments, orders, kill, …
  • Spending limits and allowed-merchants list, enforced before any browser action
  • 1-second kill switch via AbortController

Two execution rails, one interface

  • Patchright (stealth Playwright fork) for single-page billing — Anthropic, OpenAI, Vercel, Fly
  • Anthropic Computer Use for multi-step commerce — Amazon, Etsy, Shopify
  • Router picks rail by merchant. Native APIs when available.
  • Privacy.com single-use virtual cards planned as an optional liability-isolation rail

"MetaMask gave dApps signing power. termpay aims to give AI agents spending power."

Why Now

Four enablers converged in the last 12 months. None of them existed before.

Anthropic Computer Use

General-purpose visual web agent, shipped publicly Oct 2024. The first model that can actually navigate Amazon's checkout via API.

MCP critical mass

Claude Code, Cursor, Codex, Continue, Zed all ship MCP. One server, every agent — the first stable agent-to-tool interface.

Patchright (stealth automation)

Playwright fork that beats current bot-detection on real e-commerce. Without it, browser-rail wallets stay theoretical.

Protocol fragmentation

ACP, AP2, x402, Visa TAP — four competing standards. Merchants will wait before picking one. termpay can cover the gap.

System design · Security model

One local process. Card never enters the LLM context — by construction.

Designed enforcement

  • Card stored AES-256-GCM in local SQLite, key in OS Keychain
  • CVV supplied per charge via stdin/env, wiped after auth
  • Spending limits checked before the browser opens
  • Every charge requires a reason — stored and auditable

Named as risk

  • Bank chargeback protection for agent-authorized charges is ambiguous
  • Merchant ToS may forbid automated purchase; allowed-list is the lever
  • Bot-detection arms race needs ongoing maintenance
  • Privacy.com virtual cards can isolate liability per merchant
